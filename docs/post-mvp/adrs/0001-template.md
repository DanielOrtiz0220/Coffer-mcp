# 0001 - ADR Template

**Status:** template
**Date:** YYYY-MM-DD
**Deciders:** name(s)

## Context

What forced the decision? What constraints, deadlines, or prior decisions shaped the option space?

## Decision

The thing we are going to do, in one or two sentences.

## Alternatives considered

- **Option A:** Description. Rejected because reason.
- **Option B:** Description. Rejected because reason.

## Consequences

What becomes easier? What becomes harder? What new responsibilities does this create? What is the exit ramp?

## Revisit when

A trigger that should make us reopen this decision.

## Notes

This template is in `docs/post-mvp/adrs/` because ADRs created before the V1 simplification mostly described the broader product architecture. New V1 decisions can be documented near the V1 spec if they affect the private MVP directly.
